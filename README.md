基于 RK3588 的阿尔茨海默病患者智能监护设备
（不要出现任何涉及学校名称和指导老师等内容）
摘要
本作品针对阿尔茨海默病患者的照护痛点，设计了一套室内外双模式全场景智能
监护系统。室内部分以瑞芯微 RK3588 为核心，结合 USB 摄像头、MediaPipe 和
RKNN 加速引擎，实现人脸识别身份确认、人体姿态追踪、跌倒检测、门线走失
报警、久坐检测及语音提醒等功能，通过边缘 AI 实时分析患者行为。室外部分
以 ESP32 为主控，集成 GPS 定位、MPU6050 加速度计、EC6150 4G 模块和独立按
键，实现室外 GPS 定位跟踪、跌倒加速度检测、一键 SOS 求救、胸牌未佩戴检测，
并通过 MQTT 协议接入 OneNET 云平台上报位置和报警状态，同时支持 PDU 短信发
送中文报警信息至家属手机。系统通过多传感器融合策略有效降低误报率；胸牌
采用低功耗设计，MPU6050 活动检测配合超时判定未佩戴，保障续航。家属可通
过手机端实时查看位置、接收告警，也可通过短信远程查询位置。作品融合嵌入
式技术与边缘 AI，精准适配居家和外出照护需求，为患者提供全天候安全监护。 第一部分 作品概述
1.1功能与特性
系统具备室内外双模式智能监护，核心功能如下：
（1）室内人脸识别与骨骼追踪：通过 RetinaFace 检测人脸，mobilefacenet
提取特征，结合 MediaPipe 骨骼关键点，实现患者身份确认与持续追踪。即
使患者低头、背对摄像头或遮挡脸部，系统仍通过骨骼位置锁定目标，不丢
失追踪。
（2）室内跌倒检测：综合头部下落速度、人体宽高比变化、躯干倾斜角度，
采用先检测动态下落过程、再确认静态躺平的两阶段算法，避免弯腰、坐下
1
等正常动作误报，确认后自动录制现场视频。
（3）门线走失报警：用户可标定门区域，当骨骼丢失且最近门重合度高时
触发走失告警，并在室内切换为室外监护模式，自动启用室外胸牌。
（4）久坐语音提醒：通过骨骼或人脸位置判断患者静止状态，结合肩髋比
判定坐姿，久坐超过指定时间时通过麦克风播放语音提醒。
（5）室外跌倒、SOS、未佩戴检测：胸牌集成了 MPU6050，跌倒时自由落
体加冲击确认后蜂鸣器报警、发送中文短信、上报云平台；长按按键触发
SOS；指定时间无活动判定未佩戴并短信提醒家属。
（6）家属短信查询位置：家属发送任意短信到胸牌号码，系统自动回复高
德地图位置链接。
（7）四个独立按键：分别用于切换监护/待机模式，调试模式、人脸特征采
集、门线标定，操作直观。
1.2主要技术特点
（1）边缘 AI 实时推理：人脸检测与识别模型经 RKNN Toolkit 2 转换，部署
在 RK3588 NPU 上，25fps+处理 640×480 输入，实现低延迟本地化分析。
（2）骨骼追踪与面部识别融合：当患者背对或侧对摄像头时，面部识别可
能失效，系统通过 MediaPipe 骨骼关键点持续追踪患者位置和姿态，结合周
期性人脸验证，确保在遮挡场景下不丢失身份。
（3）两阶段跌倒检测：先检测头部快速下落与宽高比急剧变化作为过程触
发，再验证静态躺平姿态，严格区分正常动作与真实跌倒。
（4）室内外模式自动切换：门线走失告警后系统自动判定患者外出，停止
室内追踪并激活室外胸牌独立报警，避免室外 GPS 室内干扰。
（5）云端互联与多通道通知：通过 OneNET MQTT 上传数据，ntfy 推送和
短信双重通道确保家属及时接收告警，同时支持家属远程短信查询位置。
（6）低功耗未佩戴自检：MPU6050 定期检测三轴变化，3 指定时间无活动
判定未佩戴，自动发送提醒短信，解决认知障碍患者忘记佩戴设备的普遍问
题。

1.4主要创新点
多模态追踪锁定技术：将 MediaPipe 骨骼追踪与周期性人脸验证结合，即使患者
背对、侧对或面部部分遮挡，系统仍能通过骨骼位置持续锁定患者身份，解决了
传统人脸识别在非正面场景下失效的痛点。
室内外模式智能联动：通过门线丢失去自动切换室内/室外监护模式，室内追踪
丢失后立即启用室外胸牌定位与报警，实现无缝衔接。
未佩戴智能检测与警示：利用 MPU6050 加速度变化率结合超时判断，自动识别
患者取下胸牌并短信提醒家属，直击认知障碍患者常忘记佩戴设备的核心问题。
家属零门槛远程查询：家属发送任意短信即可获得患者位置高德地图链接，无需
安装任何 App，特别适合不擅长智能手机的老年家属使用

2.3.2 软件各模块介绍
本系统为阿尔茨海默病患者智能监护设备，软件分为室内视觉端与室外胸牌端，
自上而下分层设计，各模块独立解耦。每个模块包含流程逻辑、输入、输出变量，
室内端新增摄像头视频流、Ntfy 消息推送模块，完整实现监测、告警、远程提醒
功能。
一、室内端底层采集：摄像头视频流模块
1.功能：采集摄像头原始画面，完成图像缩放预处理，为 AI 算法提供标准图像
输入，同时输出本地可视化画面。
2.流程：初始化摄像头→持续采集图像帧→统一缩放至推理尺寸→分发图像至各
视觉检测模块；摄像头断连自动重试。
3.输入：摄像头原始数据流
4.输出：预处理标准图像帧、本地显示视频流
二、室内视觉核心：人脸识别与追踪模块
1.功能：识别人脸、比对身份、持续追踪患者，为所有室内行为检测提供目标主
体。
2.流程：每 2 帧取视频流图像→RKNN 人脸检测→提取特征与特征库比对→匹配
合格且骨骼位置对应则锁定目标→卡尔曼滤波平滑坐标；连续多帧匹配失效则退
出追踪。
3.输入：预处理图像、人体骨骼坐标、人脸特征库
4.输出：追踪状态、患者归一化坐标、人脸匹配相似度
三、室内视觉核心：室内跌倒检测模块
1.功能：基于骨骼关键点识别跌倒，过滤弯腰、下蹲等误动作。
2.流程：读取骨骼数据→计算体态、头部速度、躯干角度→多帧校验跌倒特征→
确认跌倒后输出告警信号，同步推送 Ntfy 消息。
3.输入：骨骼关键点、头部运动历史数据
4.输出：跌倒告警标志、告警时间戳、Ntfy 推送触发指令
四、室内视觉核心：门线走失检测模块
1.功能：监测患者是否走出家门警戒区域，触发走失预警。

4.2 心得体会
本项目历时三个月，从需求分析到最终联调，团队成员收获颇丰。RK3588 端的
NPU 加速是第一道难关。将训练好的 RetinaFace 和 ArcFace 模型通过 RKNN
Toolkit 转换为可执行格式时，遇到精度下降和推理错误，通过调整量化策略、
输入格式和后处理代码逐步解决。MediaPipe 在 RK3588 上 CPU 负载较高，通过
降低模型复杂度、控制帧率至 15fps 并配合多线程实现实时处理。
室外胸牌的 4G 模块（Air70E）是另一大难点。AT 指令文档存在歧义，MQTT
和短信功能需要逐条调试。尤其是 PDU 模式下 UCS2 中文编码，必须精确计算
每字节偏移，稍有差错就会发送失败。通过串口抓日志、分段调试、对比官方样
例，最终实现稳定收发。GPS 定位在室内几乎不可用，我们在代码中增加有效性
判断，避免上传零坐标。加速度计阈值经过十多次实际佩戴测试，最终确定自由
落体 0.2g、冲击 2.5g，平衡了灵敏度和误报率。
全屏无白边显示看似简单，但涉及窗口属性设置和坐标映射，门线标定中需动态
更新显示尺寸，确保全屏下鼠标位置正确。按键功能设计时，为简化操作，四个
按键分别映射模式切换、调试、特征采集和门线绘制，通过短按和长按区分功能。
久坐语音提醒使用 pyttsx3 引擎，中文发音需安装相应语音包，调试后可清晰播
报。
